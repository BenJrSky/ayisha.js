3. Event Handling: Directives vs Sub-directives

3.1 Standard Event Directives

<!-- Direct event directives -->
<button @click="count++">Increment</button>
<input @input="inputMsg = 'Typed!'" placeholder="Type here...">
<p>Input event: {{inputMsg}}</p>
<input @focus="focusMsg = 'Focused!'" @blur="focusMsg = 'Blurred!'">
<p>Focus/Blur event: {{focusMsg}}</p>
<input @change="changeMsg = 'Changed!'" placeholder="Change and blur...">
<p>Change event: {{changeMsg}}</p>
inputMsg = 'Ready'; focusMsg = 'Ready'; changeMsg = 'Ready';
Input event: {{inputMsg}}
Focus/Blur event: {{focusMsg}}
Change event: {{changeMsg}}
Tip: Use event directives to react to user actions. You can update any variable in the handler.

3.2 Comparing Standard vs Sub-directives

Ayisha.js provides two approaches to event handling: standard directives that execute expressions directly, and sub-directives that combine specific actions with trigger events. Understanding when to use each approach is crucial for building efficient applications.

<!-- Standard directives (direct expression execution) -->
<input @input="inputMsg = 'You typed: ' + event.target.value" placeholder="Type here for @input">
<div @hover="hoverMsg = 'Mouse is hovering!'" @leave="hoverMsg = 'Mouse left!'">Hover me for @hover</div>
<input @focus="focusMsg = 'Input focused with @focus!'" @blur="focusMsg = 'Input lost focus!'">

<!-- Sub-directives (action + event trigger) -->
<button @click="clickMsg = 'Set via @click at ' + new Date().toLocaleTimeString()">Click me</button>
<div @text:hover="'Text changed on hover!'" @text:leave="'Hover me'">Hover for text change</div>
<button @focus="focusMsg = 'Focus set via @focus!'">Focus me</button>

<!-- Complex example: Form with validation -->
<input @model="email" @input="validateEmail()" @class:focus="{'focused': true}" placeholder="Email">
<div @show="emailError" @class="{'error': emailError}">{{emailError}}</div>

Use standard directives when: You need complex logic, multiple operations, or custom validation on events.

⚡ Sub-directives (Event-triggered Actions)

@click: {{clickInputMsg}}

Hover me to change text

@text:hover/@text:leave: Text changes directly in the element above

Use sub-directives when: You need simple, specific actions like setting values, changing text, or toggling classes.
Performance Tip: Combine both approaches! Use sub-directives for simple UI changes and standard directives for business logic. This gives you the best performance and maintainability.
