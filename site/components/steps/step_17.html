3. Event Handling: Directives vs Sub-directives

Learn the difference between standard directives and sub-directives for efficient event handling.

3.1 Standard Event Directives

<!-- Direct event directives -->
<button @click="count++">Increment</button>
<input @input="inputMsg = 'Typed!'" placeholder="Type here...">
<p>Input event: {{inputMsg}}</p>
<input @focus="focusMsg = 'Focused!'" @blur="focusMsg = 'Blurred!'">
<p>Focus/Blur event: {{focusMsg}}</p>
<input @change="changeMsg = 'Changed!'" placeholder="Change and blur...">
<p>Change event: {{changeMsg}}</p>

3.2 Comparing Standard vs Sub-directives

Ayisha.js provides two approaches to event handling: standard directives that execute expressions directly, and sub-directives that combine specific actions with trigger events. Understanding when to use each approach is crucial for building efficient applications.

<!-- Standard directives (direct expression execution) -->
<input @input="inputMsg = 'You typed: ' + event.target.value" placeholder="Type here for @input">
<div @hover="hoverMsg = 'Mouse is hovering!'" @leave="hoverMsg = 'Mouse left!'">Hover me for @hover</div>
<input @focus="focusMsg = 'Input focused with @focus!'" @blur="focusMsg = 'Input lost focus!'">

<!-- Sub-directives (action + event trigger) -->
<button @click="clickMsg = 'Set via @click at ' + new Date().toLocaleTimeString()">Click me</button>
<div @text:hover="'Text changed on hover!'" @text:leave="'Hover me'">Hover for text change</div>
<button @focus="focusMsg = 'Focus set via @focus!'">Focus me</button>

🎮 Try it:

directInputMsg = 'Ready'; directHoverMsg = 'Ready'; directFocusMsg = 'Ready'; clickInputMsg = 'Ready'; clickFocusMsg = 'Ready'; value = ''; hoverMsg = 'Ready';

🎯 Standard Directives (Direct Execution)

@input: {{directInputMsg}}

Current value: {{value}}

Hover me for @hover

@hover/@leave: {{hoverMsg}}

Use standard directives when: You need complex logic, multiple operations, or custom validation on events.

⚡ Sub-directives (Event-triggered Actions)

@click: {{clickInputMsg}}

Hover me to change text

@text:hover/@text:leave: Text changes directly in the element above

@focus/@blur: {{clickFocusMsg}}

Use sub-directives when: You need simple, specific actions like setting values, changing text, or toggling classes.

🔍 Performance & Usage Comparison:

Standard Directives
  • @input: Complex validation logic
  • @hover: Dynamic calculations
  • @focus: Multi-step operations
  • Performance: Slower (full expression evaluation)
  • Flexibility: Maximum flexibility
  • Best for: Complex business logic
Sub-directives
  • @click: Simple value assignment
  • @text:hover: Direct text changes
  • @class:focus: CSS class toggling
  • Performance: Faster (optimized actions)
  • Flexibility: Limited but efficient
  • Best for: UI interactions & animations
💡 Pro Tip: Combine both approaches! Use sub-directives for simple UI changes and standard directives for business logic. This gives you the best performance and maintainability.

3.3 Sub-directives for Specific Events

<!-- Using sub-directives for better control -->
<div @click="status='Clicked'" @hover="status='Hovered'" @leave="status='Left'">
  Multi-event element
</div>
<button @fetch:click="'https://api.github.com/users/octocat'" @result="userData">
  Fetch on Click
</button>
<input @class:focus="{'focused': true}" @class:blur="{'focused': false}" placeholder="Dynamic classes on focus/blur">

🎮 Try it:

status = 'Ready'; userData = null; focusInput = '';

Bio: {{userData.bio}}

Public Repos: {{userData.public_repos}}

🎯 What you learned:
  • Standard directives execute expressions directly
  • Sub-directives combine actions with trigger events
  • Choose based on complexity and performance needs
  • Multiple events can be handled on the same element
