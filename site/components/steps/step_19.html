5. Advanced Loops: @for with Complex Data

Master complex data structures, nested loops, and the importance of @key for performance.

5.1 Basic Looping

<ul>
  <li @for="item in items" @key="item.id">{{item.name}}</li>
</ul>

5.2 Complex Data Structures with Nested Loops

<!-- Task Management App Example -->
<div @for="project in projects" @key="project.id" class="project-card">
  <h3>{{project.name}} ({{project.tasks.length}} tasks)</h3>
  <div @for="task in project.tasks" @key="task.id" 
       @class="{completed: task.completed, priority-high: task.priority === 'high'}">
    <label>
      <input type="checkbox" @model="task.completed">
      {{task.name}} - {{task.priority}} priority
    </label>
    <span @if="task.dueDate"> (Due: {{task.dueDate}})</span>
  </div>
</div>

5.3 Loop Performance with @key

Why @key matters:

  • Without @key: Elements are recreated when array changes, losing state
  • With @key: Elements are tracked and reused efficiently
  • Best practice: Use unique, stable values like IDs for @key
❌ Bad: @key="$index" (index changes when items move)
✅ Good: @key="item.id" (stable unique identifier)

🎮 Key Performance Demo:

itemsWithoutKey = [ {id: 1, name: 'Item 1', input: ''}, {id: 2, name: 'Item 2', input: ''}, {id: 3, name: 'Item 3', input: ''} ]; itemsWithKey = [ {id: 1, name: 'Item 1', input: ''}, {id: 2, name: 'Item 2', input: ''}, {id: 3, name: 'Item 3', input: ''} ];
