5.2. Validazione in Tempo Reale

<!-- Validazione con classi CSS dinamiche -->
<input @model="email" 
       @class="isValidEmail(email) ? 'valid' : 'invalid'"
       placeholder="Email">
<div @if="!isValidEmail(email) && email">
  Email non valida
</div>

🎮 Prova tu:

form = { username: '', email: '', password: '', confirmPassword: '', telefono: '' }; function isValidEmail(email) { const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/; return regex.test(email); } function isValidPhone(phone) { const regex = /^[+]?[\d\s\-\(\)]{8,}$/; return regex.test(phone); } function isStrongPassword(password) { return password.length >= 8 && /[A-Z]/.test(password) && /[a-z]/.test(password) && /[0-9]/.test(password); } function passwordsMatch() { return form.password === form.confirmPassword; } function isUsernameValid(username) { return username.length >= 3 && /^[a-zA-Z0-9_]+$/.test(username); } function isFormValid() { return isValidEmail(form.email) && isStrongPassword(form.password) && passwordsMatch() && isUsernameValid(form.username) && isValidPhone(form.telefono); }

Difference between @show and @if:
📱 @show: I'm always in the DOM, just hidden when false
🗑️ @if: I'm completely removed from DOM when false
🎯 What you learned:
  • @if="condition" completely removes/adds elements
  • @show="condition" just hides/shows (stays in DOM)
  • Use ! for NOT conditions
  • Ternary operator condition ? 'yes' : 'no' for text
