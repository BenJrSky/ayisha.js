Ayisha.js Directives Dictionary

@model
(Two-way binding) Binds an input element's value to a property in state with two-way reactivity. Updates the state on user input and updates the input when the state changes. Supports sub-directives for event-specific binding (e.g., @model:input, @model:change).
Use for: inputs, selects, textareas. Avoid using on non-input elements.
@click
(Event handler) Executes JavaScript code or a function when the element is clicked. Has access to state and all reactive variables.
Use for: buttons, interactive elements. Can be combined with @set or @fetch for advanced actions.
@for
(Loop) Renders the element for each item in an array or range. Syntax: item in items or (item, i) in items. Supports @key for unique identification.
Use for: lists, tables, repeated UI. Always use @key for optimal performance.

    (Conditional) Conditionally renders the element if the given expression evaluates to true. The element is removed from the DOM if false.
    Use for: conditional UI, toggles, error messages. Combine with @else or @show for alternatives.

    @show
    (Visibility) Shows or hides the element by toggling display: none based on the condition. Unlike @if, the element remains in the DOM.
    Use for: transitions, toggling visibility without removing elements from the DOM.

    @hide
    (Visibility) Hides the element by setting display: none if the condition is true. Inverse of @show.
    Use for: conditional hiding while keeping the element in the DOM.

    @fetch @result
    (Async data) Performs a fetch request to the specified URL or expression. The result is assigned to the variable named in @result. Supports sub-directives for event-based fetches (e.g., @fetch:click, @fetch:hover).
    Use for: API calls, dynamic data loading. Always pair with @result to store the response.
    @validate
    (Validation) Validates the input value using built-in or custom rules (e.g., email, required). Results are available in the _validate object.
    Use for: forms, user input. Check _validate for validation status.
    @page @link
    (SPA Routing) Enables single-page application navigation. @page marks a section as a route, @link navigates to a page without reloading.
    Use for: multi-page apps. Use _currentPage to track the active route.
    Home
    @component @src
    (Reusable HTML) Loads and renders an external HTML file as a component. @src specifies the file path. Can be combined with @page for routed components.
    Use for: modular UI, code reuse. Keep components small and focused.
    @state
    (Debug) Displays the current state object for debugging and inspection. Updates live as state changes.
    Use for: development, debugging. Remove in production.
    @log
    (Debug) Shows a log of directive actions and state changes for debugging purposes.
    Use for: tracing, debugging. Useful for learning directive flow.
    @class
    (Dynamic class) Dynamically applies CSS classes based on an expression or object. Supports sub-directives for event-based class changes (e.g., @class:focus).
    Use for: conditional styling. Combine with @style for advanced effects.
    @style
    (Dynamic style) Dynamically sets inline styles from an object or expression.
    Use for: dynamic visual changes based on state.
    @set
    (Assignment) Assigns a value to a variable or state property when an event occurs. Supports sub-directives for event-specific assignment (e.g., @set:click).
    Use for: event-driven state changes. Prefer for simple assignments over @click when no logic is needed.
    @key
    (Unique key) Provides a unique identifier for elements in @for loops to optimize DOM updates.
    Use for: lists, tables. Always use with @for for best performance.
  • @switch @case @default
    (Switch-case) Renders elements based on the value of an expression, similar to a switch-case statement. @case matches values, @default is the fallback.
    Use for: multi-state UI. Combine with @for for advanced patterns.

    Other
    @source @map @filter @reduce @initial
    (Array ops) Performs array operations directly in the template. @source provides the array, @map, @filter, @reduce transform it, @initial sets the initial value.
    Use for: data transformation, computed lists. Use @result to store the output.
    @animate
    (Animation) Applies a CSS animation or transition to the element when it enters the DOM.
    Use for: visual effects, transitions. Define animations in CSS.
    @watch
    (Watcher) Watches a variable or expression and runs code when it changes. Useful for side effects and debugging.
    Use for: logging, reacting to changes. Avoid heavy logic in watchers.
    @hover
    (Event handler) Executes code when the element is hovered. Can be used for tooltips, previews, or dynamic UI.
    Use for: interactive UI, tooltips. Combine with @class:hover for styling.
    @text
    (Dynamic text) Sets the text content of the element to the value of an expression or variable. Supports sub-directives for event-based updates.
    Use for: dynamic labels, counters. Prefer over {{ }} for one-way binding.

    Special variables

    state: Global reactive state object.
    _validate: Object with input validation results.
    _currentPage: Active page for SPA routing.

    Best practices & tips

    • Use state for all reactive variables and shared data.
    • Avoid variable names that conflict with global JS objects.
    • For validation, use @validate and check _validate for results.
    • For async fetch, use @fetch and @result on different elements.
    • For SPA, use @page and @link for navigation without reload.
    • For components, prefer reusable HTML and @src.
    • For debugging, use @state and @log to inspect state and directives.
    • Directives can be freely combined on any element.
    • Sub-directives allow advanced binding/events (e.g. @model:input).

    Advanced examples: fetch and sub-directives

    Basic fetch:
    <button @fetch="'https://jsonplaceholder.typicode.com/users'" @result="users">Load users</button>
    <ul> <li @for="user in users">{{user.name}}</li> </ul>
    Fetch on click (sub-directive):
    <button @fetch:click="'https://jsonplaceholder.typicode.com/posts'" @result="posts">Load posts</button>
    <ul> <li @for="post in posts">{{post.title}}</li> </ul>
    Fetch on hover (sub-directive):
    <div @fetch:hover="'https://jsonplaceholder.typicode.com/todos'" @result="todos" style="display:inline-block;cursor:pointer;">Hover here to load ToDo</div>
    <ul> <li @for="todo in todos">{{todo.title}}</li> </ul>
    Fetch with dynamic input (sub-directive):
    <input @model="userId" type="number" min="1" max="10" placeholder="User ID">
    <button @fetch:click="`https://jsonplaceholder.typicode.com/users/${userId}`" @result="user">Find user</button>
    <div @if="user">Name: {{user.name}}</div>
    Fetch + validation combination:
    <input @model="email" @validate="email" placeholder="Email">
    <button @fetch:click="`/api/check?email=${email}`" @result="check" @if="_validate.email">Check email</button>
    <div @if="check">Response: {{check.status}}</div>
    Multiple fetches and different @result:
    <button @fetch:click="'https://jsonplaceholder.typicode.com/users'" @result="users">Users</button> <button @fetch:click="'https://jsonplaceholder.typicode.com/posts'" @result="posts">Posts</button>
    <ul> <li @for="user in users">{{user.name}}</li> </ul> <ul> <li @for="post in posts">{{post.title}}</li> </ul>
