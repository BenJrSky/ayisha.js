2. First binding

<input @model="name" placeholder="Your name">
Hello {{name}}!
<init>
name = '';
</init>
Hello {{name}}!
Tip: You can use @model to bind any variable (e.g. name) and display it with curly braces: {{name}}.

3. Events and state

<button @click="state.count++">Increment</button>
<init>
count = 0;
</init>
Value: {{count}}
Tip: You can use state.count or count in expressions. @click lets you run any JS code on click. 
<button @click="count--">Decrement</button>
<init>
count = 0;
</init>
Value: {{count}}
Tip: You can update variables directly in event handlers.

4. Loops and conditions

<ul>
  <li @for="i in Array(counter).fill(0).map((_,i)=>i+1)">Item {{i}}</li>
</ul>
<div @if="counter > 5">Many items!</div>
<init>
counter = 10;
</init>
  • Item {{i}}
Many items!
Tip: Use @for to loop over arrays or generated lists, and @if to show/hide elements based on conditions.

5. Fetch API

<button @fetch:click="'https://jsonplaceholder.typicode.com/users'" @result="users">Load users</button>
<div @show="!users" style="color:#aaa;">No users yet, click to load</div>
<ul>
  <li @for="user in users">{{user.name}}</li>
</ul>
<init>
users = null;
</init>
No users yet, click to load
  • {{user.name}}
Tip: @fetch:click fetches data on click and stores it in users. Use @show to display messages conditionally.

6. Validation

<input type="email" @model="email" @validate="email" placeholder="Your email">
<div @show="!email" style="color:#aaa;">Enter an email to validate</div>
<div @show="email && _validate && _validate.email" style="color:#2ecc40;">Valid email!</div>
<div @show="email && _validate && !_validate.email" style="color:#ff4136;">Invalid email!</div>
<init>
email = '';
</init>
Enter an email to validate
Valid email!
Invalid email!
Tip: @validate supports built-in rules like email. The _validate object holds the validation state for each field. 
<input type="text" @model="zip" @validate="^\d{5}$" placeholder="ZIP code (5 digits)">
<div @show="!zip" style="color:#aaa;">Enter a ZIP code to validate</div>
<div @show="zip && _validate && _validate.zip" style="color:#2ecc40;">Valid ZIP!</div>
<div @show="zip && _validate && !_validate.zip" style="color:#ff4136;">Invalid ZIP!</div>
<init>
zip = '';
</init>
Enter a ZIP code to validate
Valid ZIP!
Invalid ZIP!
Tip: You can use custom regex in @validate (e.g. ^\d{5}$ for 5 digits).

7. SPA Routing

<button @link="demo">Demo</button>
<div @page="demo">Welcome!</div>
Welcome!
Welcome!
Tip: Use @link to change the current page (_currentPage), and @page to show content for a specific page.

8. Components

<component @src="/src/demo-component.html" @page="demo"></component>
<init>
_currentPage = 'demo';
</init>
Tip: <component> lets you load external HTML as a reusable component. Use @page to show it only on a specific page.

9. State and debug

<div @state></div>
<div @log></div>
Tip: @state shows the full state object. @log prints debug info for the current context.
